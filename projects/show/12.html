Intranet Search Engine


A search engine is a very common idea. What's different about this one? Well, I wished once (you might not agree with me) that I had a search engine that could cache important documents from bitmesra.ac.in, and bitbus.in, and anything BIT related, say like a notice you're searching, or a form that you want to find, was all offline. Just visit that website, and ask something like "Amrit Dhara phone number" or "academic transcript form" or ...

This project is about creating a simple search engine that does that. We will host it on the intranet afterwards, and I think it would be a great opportunity to learn how complex websites work. And who knows, if the website is good enough, we could ask Aman (:P) to help with some of his fancy ML tricks.

Start Date: around mid October
Prerequisites: understand how websites work
Skills (not required, but we'll learn along the way): Django / Ruby on Rails / PHP



Time Needed => 3 months
Status => not started
